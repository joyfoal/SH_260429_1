# PRODOO.AI 랜딩 페이지 기술명세서

## 1. 문서 개요

본 문서는 `PRODOO.AI` 싱글 페이지 랜딩 웹사이트의 현재 구현 기준 기술명세서입니다. 대상 서비스는 AI 에이전트 기반 앱 개발 에이전시를 소개하고, 방문자가 서비스 가치를 이해한 뒤 상담 또는 문의 행동으로 전환하도록 설계된 정적 웹페이지입니다.

- 프로젝트명: `PRODOO.AI Landing Page`
- 페이지 유형: 정적 싱글 페이지 랜딩
- 구현 방식: Vanilla HTML, CSS, JavaScript
- 주요 목적: 서비스 소개, 신뢰 형성, 무료 상담 및 고객 문의 유도
- 배포 형태: 정적 호스팅 또는 로컬 브라우저 실행

## 2. 대상 사용자

- 바이브코딩 또는 AI 기반 개발에 관심 있는 입문자
- 빠르게 MVP를 만들고 싶은 개인, 초기 창업팀, 소규모 조직
- 개발 지식은 많지 않지만 앱 개발 또는 업무 자동화를 검토하는 PM, 기획자, 운영 담당자
- 기존 서비스 개선, 유지보수, 자동화 도입을 원하는 비개발 중심 팀

## 5. 기능 요구사항

### 5.1 내비게이션

- 상단 헤더는 화면 상단에 sticky 형태로 고정됩니다.
- 메뉴 클릭 시 같은 페이지의 대상 섹션으로 스크롤 이동합니다.
- 주요 메뉴는 `가치`, `프로세스`, `사례`, `FAQ`로 구성됩니다.
- 브랜드 클릭 시 Hero 섹션으로 이동합니다.

### 5.2 다국어

- 지원 언어는 한국어(`ko`), 영어(`en`), 일본어(`ja`)입니다.
- 최초 언어는 브라우저 언어를 기준으로 자동 감지합니다.
- 사용자가 언어 버튼을 클릭하면 즉시 페이지 텍스트가 변경됩니다.
- 선택한 언어는 `localStorage`의 `prodoo.lang` 키에 저장합니다.
- 다국어 적용 대상은 본문 텍스트, placeholder, aria-label, document title, meta description, Open Graph title/description/locale입니다.
- 지원하지 않는 언어는 한국어를 기본값으로 사용합니다.

### 5.3 테마 전환

- 지원 테마는 라이트(`light`)와 다크(`dark`)입니다.
- 최초 테마는 저장값이 있으면 저장값을 사용하고, 없으면 OS의 `prefers-color-scheme` 값을 따릅니다.
- 사용자가 테마 버튼을 클릭하면 `html[data-theme]` 속성이 변경됩니다.
- 선택한 테마는 `localStorage`의 `prodoo.theme` 키에 저장합니다.
- 저장소 접근이 차단된 환경에서도 기본 테마로 동작해야 합니다.

### 5.4 Scroll Reveal

- `.reveal` 클래스를 가진 섹션은 뷰포트 진입 시 `.is-visible` 클래스를 부여받습니다.
- 브라우저가 `IntersectionObserver`를 지원하지 않으면 모든 reveal 요소를 즉시 표시합니다.
- 사용자가 모션 감소 설정을 사용 중이면 애니메이션 없이 콘텐츠를 바로 표시합니다.

### 5.5 FAQ 아코디언

- 각 FAQ 질문은 버튼으로 구현합니다.
- 질문 클릭 시 연결된 답변 패널의 `hidden` 상태를 토글합니다.
- 버튼의 `aria-expanded` 값은 열림/닫힘 상태와 동기화합니다.
- 여러 FAQ 항목은 동시에 열릴 수 있습니다.

### 5.6 CTA 이벤트 훅

- `data-cta` 속성이 있는 링크를 클릭하면 `prodoo:cta-click` 커스텀 이벤트를 발생시킵니다.
- 이벤트 상세 정보에는 CTA label과 timestamp를 포함합니다.
- 현재 분석 도구는 연결되어 있지 않으며, 향후 연동을 위한 안정적인 이벤트 훅으로 사용합니다.

### 5.7 고객 문의 폼

- 문의 폼 필드는 이름, 이메일, 문의 내용으로 구성합니다.
- 문의 내용은 필수 입력값입니다.
- 이메일 형식 검증은 브라우저 기본 `input[type="email"]` 검증을 사용합니다.
- 제출 시 서버로 전송하지 않고 `mailto:joyfoal@naver.com` 링크를 생성합니다.
- 메일 제목에는 현재 언어별 subject prefix와 이름을 포함합니다.
- 메일 본문에는 이름, 이메일, 문의 내용, 웹페이지 전송 문구를 포함합니다.
- 제출 전 `checkValidity()`를 실행하고, 유효하지 않으면 `reportValidity()`를 호출합니다.
- 정상 제출 시 `prodoo:contact-submit` 커스텀 이벤트를 발생시킵니다.

## 6. 비기능 요구사항

### 6.1 접근성

- 시맨틱 HTML 태그인 `header`, `nav`, `main`, `section`, `footer`를 사용합니다.
- 본문 바로가기 링크를 제공합니다.
- 주요 상호작용 요소는 키보드로 조작 가능해야 합니다.
- 포커스 상태는 시각적으로 구분되어야 합니다.
- FAQ 버튼은 `aria-expanded`를 통해 상태를 전달해야 합니다.
- 테마 및 언어 버튼은 `aria-pressed`를 통해 선택 상태를 전달해야 합니다.
- 모션 감소 환경에서는 애니메이션을 비활성화합니다.

### 6.2 반응형

- 기본 레이아웃은 모바일 우선으로 구성합니다.
- `768px` 이상에서 Hero 영역은 2열 레이아웃을 사용하고 카드 그리드는 2열로 전환합니다.
- `1024px` 이상에서 카드 그리드는 3열로 전환합니다.
- 컨테이너 최대 너비는 `1200px`입니다.
- 작은 화면에서도 헤더 우측 컨트롤은 줄바꿈이 가능해야 합니다.

### 6.3 성능

- 외부 JavaScript 라이브러리를 사용하지 않습니다.
- 페이지는 정적 파일만으로 동작해야 합니다.
- 애니메이션은 opacity와 transform 중심으로 구현합니다.
- 이미지 의존성은 현재 없으며, 추가 시 WebP 및 lazy loading 사용을 권장합니다.
- 목표 Lighthouse Performance 점수는 90점 이상입니다.

### 6.4 SEO

- 페이지는 단일 H1을 유지합니다.
- 각 주요 섹션은 H2를 사용합니다.
- `title`, `meta description`, Open Graph title, Open Graph description, Open Graph locale을 제공합니다.
- 언어 변경 시 메타 정보도 함께 갱신합니다.

### 6.5 브라우저 호환성

- 최신 Chromium, Safari, Firefox를 기준으로 합니다.
- `IntersectionObserver`, `localStorage`, `CustomEvent`, `FormData`, optional chaining을 사용하는 최신 브라우저 환경을 전제로 합니다.
- `IntersectionObserver` 미지원 환경에는 콘텐츠 즉시 표시 fallback을 제공합니다.

## 7. 디자인 명세

### 7.1 디자인 방향

- 미니멀하고 따뜻한 중성 톤을 사용합니다.
- 높은 가독성과 넉넉한 여백을 우선합니다.
- CTA와 강조 요소에는 부드러운 주황 계열 포인트 컬러를 사용합니다.
- 카드, 버튼, 폼 요소는 둥근 모서리와 얕은 구분감을 사용합니다.

### 7.2 디자인 토큰

- Background: `#f7f5f2`
- Surface: `#ffffff`
- Primary Text: `#1f1f1f`
- Secondary Text: `#5b5b5b`
- Accent: `#d97757`
- Accent Soft: `#f2d8cf`
- Border: `#e8e1db`
- Radius: `14px`
- Container: `1200px`
- Spacing scale: `8px`, `16px`, `24px`, `32px`, `48px`, `72px`, `96px`

### 7.3 다크 테마

- `html[data-theme="dark"]`에서 색상 토큰을 재정의합니다.
- 다크 배경은 `#0b0d10`, 표면은 `#10141b`를 사용합니다.
- 텍스트와 border, shadow, accent soft 값은 다크 환경에 맞게 조정합니다.

## 8. 파일 구조

```text
.
├── README.md
├── TECHNICAL_SPEC.md
├── index.html
├── script.js
└── styles.css
```

### 8.1 `index.html`

- 전체 페이지 구조를 정의합니다.
- 각 텍스트 요소는 `data-i18n` 키로 다국어 사전과 연결합니다.
- placeholder는 `data-i18n-placeholder`, aria-label은 `data-i18n-aria`로 연결합니다.
- 문의 폼과 CTA 링크의 기본 마크업을 포함합니다.

### 8.2 `styles.css`

- 디자인 토큰, 레이아웃, 컴포넌트 스타일을 정의합니다.
- 라이트/다크 테마를 CSS 커스텀 프로퍼티로 관리합니다.
- 반응형 breakpoint와 모션 감소 설정을 처리합니다.

### 8.3 `script.js`

- 다국어 사전과 i18n 적용 로직을 포함합니다.
- 언어 및 테마 상태를 `localStorage`와 UI에 동기화합니다.
- reveal 애니메이션, FAQ 아코디언, CTA 이벤트, 문의 폼 제출 로직을 처리합니다.

## 9. 데이터 및 상태

### 9.1 로컬 저장소

- `prodoo.lang`: 사용자가 선택한 언어
- `prodoo.theme`: 사용자가 선택한 테마

### 9.2 커스텀 이벤트

- `prodoo:cta-click`
  - 발생 시점: CTA 링크 클릭
  - detail: `label`, `timestamp`

- `prodoo:contact-submit`
  - 발생 시점: 문의 폼 유효성 검증 통과 후 mailto 이동 직전
  - detail: `lang`, `hasEmail`, `hasName`

## 10. 제약사항 및 리스크

- 문의 폼은 `mailto:` 방식이므로 사용자 기기에 메일 클라이언트가 설정되어 있어야 합니다.
- 문의 내용은 서버에 저장되지 않습니다.
- 제출 성공 여부, 메일 발송 실패 여부, 재시도 로직은 제공하지 않습니다.
- 실제 고객 후기와 성과 데이터는 현재 정적 카피로 구성되어 있어, 운영 전 실제 데이터로 교체하는 것이 좋습니다.
- 분석 도구는 연결되어 있지 않으며 커스텀 이벤트 훅만 제공합니다.

## 11. 검수 기준

- 모든 주요 섹션이 의도한 순서로 표시됩니다.
- 한국어, 영어, 일본어 전환 시 텍스트와 메타 정보가 정상 변경됩니다.
- 라이트/다크 테마 전환 및 새로고침 후 저장 상태가 유지됩니다.
- FAQ 클릭 시 답변 패널과 `aria-expanded` 값이 함께 변경됩니다.
- 문의 내용 필수 입력 검증이 동작합니다.
- 유효한 문의 제출 시 수신자 `joyfoal@naver.com`으로 `mailto:`가 생성됩니다.
- 모바일, 태블릿, 데스크톱에서 레이아웃이 깨지지 않습니다.
- 키보드 탐색과 포커스 표시가 가능합니다.
- 모션 감소 설정에서 reveal 애니메이션이 비활성화됩니다.
